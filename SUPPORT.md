# Support

Ce module ZM40 est gratuit et open source (voir le fichier LICENSE). Il est offert à la communauté PrestaShop dans l'esprit du partage, sans support garanti. Voici comment obtenir de l'aide selon votre besoin.

## Signaler un bug

Ouvrez une issue sur GitHub. Pour qu'elle soit utile, indiquez :

- la version de PrestaShop ;
- la version de PHP ;
- la version du module ;
- les étapes pour reproduire le problème ;
- le message d'erreur ou les logs, si vous en avez.

Les bugs reproductibles sont traités au mieux, selon le temps disponible, sans garantie de délai. C'est de l'open source : votre rapport, et vos correctifs, aident toute la communauté.

## Proposer une idée ou une amélioration

Ouvrez une issue pour suggérer une fonctionnalité. Les pull requests sont les bienvenues ; merci de décrire votre contexte et d'ouvrir une issue avant les changements importants.

## Besoin d'aide concrète ?

L'installation, la configuration sur votre boutique, l'adaptation à votre thème, le debug spécifique à votre environnement et le conseil ne relèvent pas du support gratuit : c'est mon métier, et c'est ce qui fait vivre ces modules offerts.

Pour une aide rapide et fiable, et c'est moi qui réponds : zm40.com — intervention à l'heure ou au forfait. On lit, on diagnostique, on corrige.

C'est aussi le bon canal pour une fonctionnalité sur-mesure, une intégration (ERP, marketplace, API, transporteurs) ou un accompagnement sur votre boutique.

## En résumé

| Votre besoin | Où aller |
|---|---|
| Signaler un bug | Issue GitHub |
| Proposer une idée ou une PR | Issue GitHub |
| Aide install / config / adaptation | zm40.com (sur devis) |
| Dev sur-mesure, intégration | zm40.com (sur devis) |

Le code est offert. Mon temps, lui, fait vivre tout ça. Merci de votre compréhension.
